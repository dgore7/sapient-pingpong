Sapient Ping-Pong

An Interactive Ping-Pong Platform

View project on GitHub

Introduction

Sapient Pingpong is an instrumented pingpong platform developed by the SapientNitro Chicago 2016 summer interns, Wesley Hampson and David Gorelik. The platform is built with nodejs and express on the back-end, and React on the front-end. It also includes a serial reader wrtiten in C# that reads input from an RFID scanner.

Implementation

Project Structure

The project is divided into two parts, the RFID reader and webapp. Through out the project we follow an MVC pattern in each piece of the app. The RFID folder contains the C# program that runs on a windows IoT operating system and reads input from a pcProx RFID reader. The webapp folder contains an API that facilitates communication with the database, a server folder which handles page requests, a public folder containing assets, css, javascript, and transpiled react code, and the src folder contains the original react source code

How it works

The scoreboard is a react app designed for a tablet screen. The app hangs over the table and keeps score of the game through user button presses. The buttons fixed underneath the table are made by Flic. Flic buttons work through bluetooth, and come with a phone app which can be configured to recieve button presses, and send an http request to the server based on the particular type of press. The button contact may be removed and used with the preferred button type (we recommend arcade style). The buttons also have other functionality such as point removal (on button hold) and server tracking (hold button at the start of game). The scoreboard also has a digital restart and takeback option. If server tracking is enabled, The serving marker will switch every five serves untill a player reaches 20. At that point the opposite player will serve untill the score is equalized, and then the server will change every two serves until a player wins by two poiints. On winning, the winners player card will be focused on the screen, and after 5 seconds the scoreboard will return to its idle state. We also used pusher as a socket for sending and recieving score and login information on the client.
